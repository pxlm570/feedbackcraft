# 智能绩效反馈生成器 — 方案说明

## 一、问题诊断

绩效反馈是管理者最频繁的HR工作之一，但长期面临三个核心痛点：

**时间成本高。** 管理者平均需要30-60分钟撰写一份结构完整、内容具体的绩效反馈。在季度评估季，一位管理5人团队的经理仅写反馈就需要半天时间，严重影响核心业务工作。

**质量参差不齐。** 不同管理者的反馈质量差异巨大。经验丰富的管理者能写出有深度的反馈，但多数管理者的输出停留在"工作态度积极""团队协作好"等模糊套话，缺乏具体事例和可执行的改进建议。这让员工感到反馈不专业、不公平。

**AI工具的局限。** 虽然已有通用AI工具可以辅助写作，但它们缺乏绩效管理的领域知识——不知道反馈应包含哪些结构、如何平衡正负评价、如何写出SMART标准的发展建议。直接使用通用AI生成的反馈往往结构混乱、用词不当。

## 二、方案设计

我们的核心创新是：**将绩效反馈的专业流程封装为一个AI Agent Skill**，而非构建一个独立的Web应用。

**Skill架构。** 整个方案的核心是一个SKILL.md文件，包含8个部分：概述、使用场景、输入采集、输出结构、语气指南、质量规则、常见错误和完整示例。这个文件定义了AI在生成绩效反馈时应遵循的完整流程和质量标准。

**输入输出设计。** 输入端采用结构化引导：AI主动询问员工姓名、岗位、评估周期、2-5条具体成就、1-3条改进点。输出端采用5部分模板：总体评价、核心优势、发展建议（SMART格式）、下一步行动。这种"结构化输入→结构化输出"的设计确保了反馈质量的一致性。

**语气系统。** 提供三种语气风格：正式（用于HR系统和晋升评估）、温和（用于初级员工和辅导场景）、直接（用于资深员工和数据驱动的评估）。用户可根据场景选择，AI自动调整措辞和框架。

**零依赖设计。** 单个Markdown文件，无需服务器、数据库或npm包。复制到技能目录即可使用，数据全程在用户本地处理，不上传任何外部服务。

## 三、AI工具选型

选择Claude Code和Codex作为主要运行平台，理由如下：

**复用现有生态。** 管理者日常已在使用AI编程助手，Skill可以直接嵌入其工作流，无需打开新应用。这比独立Web应用的使用门槛更低。

**隐私保护。** 企业绩效数据高度敏感。Skill在本地AI助手中处理，数据不经过任何第三方服务器，满足企业数据安全要求。

**可扩展性。** 基于通用Skill格式，任何开发者都可以fork并适配自己的绩效管理框架、企业文化和评估标准。

**为什么不选Web应用？** 独立Web应用需要服务器、用户账号、数据存储，增加了部署复杂度和数据泄露风险。Skill方案以最小的交付物实现了相同的功能价值。

## 四、迭代记录

**v1 — 基础模板。** 最初版本仅包含输出模板（5部分结构），AI按模板填充内容。测试发现：缺乏输入引导，用户提供的信息不够具体，导致输出仍然泛泛而谈。

**v2 — 结构化输入。** 增加了输入采集模块，AI主动询问必填信息，并提供"好例子vs坏例子"对比引导。测试发现：输出质量提升，但不同场景的语气不匹配。

**v3 — 语气系统。** 增加了三种语气风格（正式/温和/直接），每种风格有独立的措辞指南和示例开头。测试发现：AI偶尔会编造数据或使用模糊措辞。

**v4 — 质量规则和反模式。** 增加了7条质量规则和常见错误表，明确禁止编造数据、模糊措辞等行为。加入完整输入→输出示例作为参考模式。当前版本。

## 五、效果评估

**时间节省。** 从30-60分钟缩短至3-5分钟（含AI生成和人工审核），效率提升约10倍。

**质量一致性。** 所有输出遵循统一的5部分结构和质量规则，消除了管理者之间的质量差异。

**专业性提升。** 内置HR最佳实践框架（SMART建议、2:1正负比、具体事例引用），产出质量达到专业HR水平。

**局限性。** 当前版本依赖用户提供具体事例，如果用户无法回忆关键事件，输出质量会下降。未来可考虑接入项目管理工具（如Jira、飞书）自动提取关键事件。

**未来方向。** 支持更多反馈场景（360度反馈、项目复盘、入职评估）；接入企业绩效系统实现一键导入导出；增加多语言支持。
