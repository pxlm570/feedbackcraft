内置质量规则

✓每句反馈必须引用用户提供的具体事例
✓禁止使用"工作态度积极"等模糊措辞
✓正负评价保持 2:1 比例
✓发展建议必须符合 SMART 原则
✓绝不编造数据或成就
✓信息不足时主动询问而非猜测
Design Philosophy

设计理念

为什么我们选择做 Skill，而不是又一个独立应用

🔒

数据不出本地

绩效数据是企业最敏感的信息之一。传统 Web 应用需要将数据上传到服务器，而 Skill 在本地 AI 助手中处理，数据永远不离开你的设备。

🔄

嵌入工作流

最好的工具是"无处不在"的。通过 Skill 格式，FeedbackCraft 嵌入到开发者日常使用的 AI 编程助手中，无需额外学习成本，打开对话即可使用。

⚙

开放可定制

基于 Markdown 的 Skill 格式，任何开发者都可以 fork、修改、适配自己的绩效管理框架和企业文化。零依赖，零成本。

为什么不是独立 Web 应用？

独立应用的问题

  • ✕需要服务器和域名，增加维护成本
  • ✕用户数据上传到第三方，隐私风险
  • ✕需要注册账号，增加使用门槛
  • ✕功能更新需要重新部署

Skill 的优势

  • ✓单个文件，复制即用
  • ✓数据本地处理，零隐私风险
  • ✓在现有 AI 助手中直接调用
  • ✓修改配置文件即可适配
Open Source Skill

Skill 开源

将完整 AI 能力安装到你的编程助手中

什么是 Skill？

Skill 是一个 Markdown 配置文件，定义了 AI 助手在特定场景下的行为规范。安装后，AI 会按照 Skill 中的流程和质量标准来生成绩效反馈。

# Codex

使用方式

1 从 GitHub 下载 SKILL.md 文件
2 放入对应 AI 工具的 skills 目录
3 在对话中说"帮我写一份绩效反馈"即可触发
