Problem

我们发现了什么

绩效反馈是管理者最耗时的HR工作之一，但产出质量往往不尽人意

⏲

耗时费力

管理者平均花费 30-60 分钟 撰写一份绩效反馈，季度评估期间严重影响核心业务工作。

📈

质量参差

不同管理者写出的反馈质量差异巨大，缺乏统一标准，员工感到 不公平、不专业。

😐

泛泛而谈

"工作态度积极""团队协作好" — 大量反馈停留在 模糊套话，缺乏具体事例和可执行的改进建议。

👨‍🎓

缺乏HR专业性

业务管理者擅长专业技能，但 不熟悉绩效管理框架，难以产出符合HR规范的反馈文本。

Solution

我们的解决方案

不是又一个独立应用，而是一个可复用的 AI Agent Skill

⚡

创新点：AI Agent Skill

我们将绩效反馈的专业流程封装为一个 Skill 文件（SKILL.md），可直接安装到 Claude Code、Codex 等通用 AI 编程助手中。用户无需打开新应用，在日常编程环境中即可调用。

📝

结构化输入采集

引导用户逐步提供：员工信息、具体成就、改进点。确保反馈有据可依。

💡

专业输出生成

基于 HR 最佳实践框架，生成5部分结构化反馈：总评、优势、改进、发展建议、下一步。

🎨

语气风格定制

三种语气可选：正式（HR系统）、温和（初级员工）、直接（资深员工），适配不同场景。

How It Works

使用流程

4 步完成，从安装到产出专业反馈

1

安装 Skill

将 SKILL.md 文件放入 ~/.claude/skills/ 或 ~/.codex/skills/ 目录，重启 AI 助手即可生效。

2

提供上下文

告诉 AI：员工姓名、岗位、评估周期、具体成就和改进点。AI 会引导你逐步完善信息。

3

AI 生成反馈

AI 按照专业框架生成结构化反馈文本，包含总评、优势分析、改进建议和发展规划。

4

审核与调整

审阅生成结果，可要求 AI 调整语气、补充细节或修改措辞，直到满意为止。

Technology

技术架构

轻量、开放、隐私优先

📦 跨平台兼容

基于 Claude Code / Codex 通用 Skill 格式，一个文件适配多个 AI 编程助手。安装即用，无需额外配置。

🔒 数据隐私

所有数据处理在用户本地 AI 助手中完成，无需上传到任何服务器。员工信息不会离开你的设备。

🚀 零依赖部署

单个 Markdown 文件，无服务器、无数据库、无 npm 包。复制到技能目录即可使用。

🔄 开放可扩展

基于 Skill 格式，任何开发者都可以 fork、修改、适配自己的绩效管理框架和企业文化。
